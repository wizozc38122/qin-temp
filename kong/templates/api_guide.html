📖 Kong API 快速指引

這是一份供您在 Qin Kong UI 中快速建立 JSON Payload 的參考手冊。請複製以下 JSON 結構並在「建立」或「編輯」區域中使用。

📦 1. Service (服務出口)

定義後端目標應用程式。可以直接用 `url` 簡寫，或是拆分 `host`, `port`, `path`。

{
  "name": "payment-api",
  "url": "http://internal-payment.local:8080/api",
  "retries": 5,
  "connect_timeout": 60000,
  "write_timeout": 60000,
  "read_timeout": 60000
}

🛤️ 2. Route (路由入口)

定義外部如何存取 Service。最常用 `paths` 或 `hosts` 進行分流。

{
  "name": "payment-route",
  "paths": ["/v1/pay"],
  "methods": ["GET", "POST"],
  "strip_path": true,
  "preserve_host": false
}
strip_path: 若為 true，轉發到 Service 時會去掉 /v1/pay。

🔌 3. Plugin (擴充套件)

在 Service 或 Route 上加上功能。例如：Rate Limiting (限流)。

{
  "name": "rate-limiting",
  "config": {
    "minute": 20,
    "hour": 500,
    "policy": "local"
  }
}

⚠️ 註：目前 UI 的「一鍵建立」區僅限 Service/Route，Plugin 請透過 Kong Admin API 直連呼叫。

⚖️ 4. Upstream & Target (負載均衡)

取代單一的 Service URL，建立一個 Upstream 群組以實現 Load Balancing。

// 1. 建立 Upstream
{
  "name": "payment-upstream"
}

// 2. 新增 Target (節點)
{
  "target": "192.168.1.10:8000",
  "weight": 100
}

建立 Upstream 後，Service 的 `host` 即可填寫 `payment-upstream`。
