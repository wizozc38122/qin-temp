📖 Kong API 說明

這是一份供您快速參考的 API 手冊。以下列出目前 Qin Kong UI 串接 Kong 伺服器所使用的實際 API Endpoints，方便您在需要時透過指令（如 curl 或 Postman）自行操作。

📦 1. Services (服務) API

管理後端目標應用程式。可以直接用 `url` 簡寫，或是拆分 `host`, `port`, `path`。

取得所有 Service 
GET /services?size=1000
建立 Service 
POST /services
{
  "name": "my-service",
  "url": "http://internal.api:8080"
}
更新 Service 
PATCH /services/{service_id}
{ "retries": 5 }
刪除 Service 
DELETE /services/{service_id}

🛤️ 2. Routes (路由) API

定義外部如何存取 Service。最常用 `paths` 或 `hosts` 進行分流。

取得所有 Route 
GET /routes?size=1000
建立 Route (綁定特定 Service) 
POST /services/{service_id}/routes
{
  "name": "my-route",
  "paths": ["/v1/api"],
  "strip_path": true
}
更新 Route 
PATCH /routes/{route_id}
{ "methods": ["GET", "POST"] }
刪除 Route 
DELETE /routes/{route_id}

⚙️ 3. Node & System API

取得 Kong 節點的基本資訊與健康狀態。

節點資訊 
GET /

回傳 Kong 版本、Plugins 清單與設定狀態。

健康狀態 (Status) 
GET /status

回傳資料庫連線與記憶體等資源使用狀況。
